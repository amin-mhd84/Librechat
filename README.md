## 📖 معرفی پروژه

**LibreChat** یک چت‌بات پرسش‌وپاسخ است که با استفاده از معماری **RAG (Retrieval-Augmented Generation)** ساخته شده و به‌جای تکیه بر دانش از‌پیش‌آموزش‌دیده‌ی یک مدل زبانی بزرگ، پاسخ‌های خود را از میان مجموعه‌ای از اسناد واقعی بازیابی می‌کند.

موضوع دانشی این ربات حول محور **فلسفه‌ی لینوکس، نرم‌افزار آزاد (Free Software) و چهره‌های تأثیرگذار این حوزه** مثل **لینوس توروالدز** و **ریچارد استالمن** است. داده‌های آن از منابعی با فرمت‌های گوناگون (کتاب PDF، صفحات وب، مقالات ویکی‌پدیا و فایل‌های HTML) جمع‌آوری و در یک خط‌لوله‌ی کامل RAG پردازش شده‌اند.


---

## ✨ ویژگی‌ها

- 🔍 **بازیابی معنایی (Semantic Retrieval)** از میان اسناد چندزبانه (فارسی و انگلیسی) با استفاده از مدل‌های تعبیه‌سازی (Embedding)
- 📄 **بارگیری داده از منابع ناهمگون**: PDF فارسیِ اسکن‌نشده، صفحات وب، مقالات ویکی‌پدیا و فایل‌های HTML استخراج‌شده با Web Scraping
- ✂️ **تقسیم‌بندی هوشمند اسناد** (Chunking) برای بهبود کیفیت بازیابی
- 🧠 **زنجیره‌ی پرسش‌وپاسخ LangChain** با قابلیت اتصال به مدل‌های زبانی مختلف (OpenAI، DeepSeek، Groq، Cohere و ...)
- 🗂️ **مخزن برداری (Vector Store)** مبتنی بر Chroma برای ذخیره و جست‌وجوی سریع اسناد
- 📐 **خروجی ساختاریافته**: پاسخ‌های مدل به‌صورت عبارات کوتاه و قابل داوری خودکار تجزیه می‌شوند
- 🔐 **مدیریت امن کلیدهای API** با فایل `.env` (کلیدها هرگز داخل کد قرار نمی‌گیرند)

## 🏗️ معماری پروژه (RAG Pipeline)

```
┌──────────────┐     ┌───────────────┐     ┌──────────────────┐     ┌─────────────────┐
│  منابع داده   │ ──▶ │  بارگیری اسناد │ ──▶ │  تقسیم‌بندی متن   │ ──▶ │  تعبیه‌سازی برداری │
│ PDF/Web/Wiki  │     │  (Loaders)    │     │   (Splitters)    │     │   (Embeddings)   │
└──────────────┘     └───────────────┘     └──────────────────┘     └────────┬─────────┘
                                                                              │
                                                                              ▼
┌──────────────┐     ┌───────────────┐     ┌──────────────────┐     ┌─────────────────┐
│  پاسخ نهایی    │ ◀── │ تجزیه‌ی خروجی  │ ◀── │  مدل زبانی (LLM) │ ◀── │ مخزن برداری Chroma │
│ (فرمت ساختاریافته)│     │ (Output Parser)│     │ + دستور سفارشی   │     │  (Vector Store)  │
└──────────────┘     └───────────────┘     └──────────────────┘     └─────────────────┘
```

1. **بارگیری (Load):** هر منبع داده با لودر مخصوص خودش به‌شکل `Document` استخراج می‌شود.
2. **تقسیم‌بندی (Split):** اسناد بزرگ به قطعات کوچک‌تر شکسته می‌شوند تا بازیابی دقیق‌تر شود.
3. **تعبیه‌سازی (Embed):** هر قطعه به یک بردار عددی چندبعدی تبدیل می‌شود.
4. **ذخیره‌سازی (Store):** بردارها در یک پایگاه‌داده‌ی برداری (Chroma) ذخیره می‌شوند.
5. **بازیابی (Retrieve):** با ورود پرسش کاربر، مرتبط‌ترین قطعات از مخزن برداری واکشی می‌شوند.
6. **تولید پاسخ (Generate):** مدل زبانی با دریافت پرسش + اسناد بازیابی‌شده، پاسخ نهایی را در قالب موردنظر تولید می‌کند.

## 🚀 راه‌اندازی و اجرا

### پیش‌نیازها
- Python ≥ 3.10
- حساب کاربری و کلید API از حداقل یکی از سرویس‌های مدل زبانی (OpenAI، DeepSeek، Groq، Cohere، Google یا HuggingFace)

### مراحل نصب

```bash
# ۱. کلون کردن مخزن
git clone https://github.com/USERNAME/librechat.git
cd librechat

# ۲. ساخت محیط مجازی
python -m venv venv
source venv/bin/activate        # ویندوز: venv\Scripts\activate

# ۳. نصب کتابخانه‌ها
pip install -r requirements.txt

# ۴. تنظیم کلیدهای API
cp .env.example .env
# سپس فایل .env را باز کرده و کلیدهای خودتان را وارد کنید
```

### اجرای نوت‌بوک

```bash
jupyter notebook librechat.ipynb
```
سلول‌ها را به‌ترتیب از بالا به پایین اجرا کنید. مراحل اصلی به‌ترتیب عبارت‌اند از: بارگیری داده → تقسیم‌بندی اسناد → تعبیه‌سازی و ذخیره در مخزن برداری → ساخت زنجیره‌ی پرسش‌وپاسخ.

## 💬 نحوه‌ی استفاده

پس از اجرای کامل نوت‌بوک و ساخت زنجیره‌ی RAG، کافی است پرسش خود را به‌عنوان ورودی به زنجیره بدهید:

```python
question = "ریچارد استالمن چه سالی بنیاد نرم‌افزار آزاد را تأسیس کرد؟"
response = rag_chain.invoke({"question": question})
print(response)
```

خروجی مدل به‌شکل عبارات کوتاه و ساختاریافته (نه جملات کامل) بازگردانده می‌شود تا امکان داوری خودکار پاسخ‌ها فراهم شود.

## 🔐 نکته‌ی امنیتی

- هرگز کلیدهای API را مستقیم داخل کد یا نوت‌بوک قرار ندهید.
- کلیدها را همیشه در فایل `.env` نگه دارید (این فایل در `.gitignore` قرار دارد و آپلود نمی‌شود).
- اگر کلیدی به‌اشتباه در تاریخچه‌ی گیت قرار گرفت، آن را فوراً از پنل سرویس مربوطه غیرفعال (Revoke) کنید — تغییر بعدی در کد کافی نیست، چون تاریخچه‌ی گیت آن را همچنان نگه می‌دارد.
